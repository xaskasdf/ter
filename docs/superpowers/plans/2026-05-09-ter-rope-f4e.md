# `ter` RoPE Kernel Plan (F4-cont)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development.

**Goal:** Implement `tk_rope` — Rotary Position Embedding — as a pure-SIMD ternary kernel. End state: random query/key vector → RoPE rotation via kernel → matches numpy reference within bounded relative error.

**Architecture:** **Pure-SIMD kernel, host-prepared inputs.** RoPE is paired rotation: `(x[2k], x[2k+1]) → (x[2k]*cos - x[2k+1]*sin, x[2k]*sin + x[2k+1]*cos)`. The kernel sees three pre-built inputs:

- `cos_vec`: `cos[k]` duplicated at lanes `(2k, 2k+1)`
- `sin_vec`: `sin[k]` duplicated at lanes `(2k, 2k+1)`
- `rotated_x`: pair-swap with negation, lanes `(2k, 2k+1) = (-x[2k+1], x[2k])`

Then the kernel does `y = x * cos_vec + rotated_x * sin_vec` in 2 `tvmul` + 1 `tvadd`. Counter signature is exact: 4 `tvload`, 2 `tvmul`, 1 `tvadd`, 1 `tvstore`. The host builds `cos_vec`, `sin_vec`, `rotated_x` at runtime (frequencies and pair-swap are cheap).

## Why pure-SIMD

The alternative — kernel computes pair-swap and frequency lookup internally — needs new opcodes (`TVSWAP_PAIRS`) or extensive software loops. The pure-SIMD path keeps the kernel tiny (~10 instructions) and demonstrates the rotation math cleanly. Cost: more host setup code, but it's all simple linear math.

For real attention, the host loop over (batch, head, position) tuples is going to do this anyway — RoPE prep is per-token-per-head host work. The kernel is the per-token compute.

## Task F4.9.1 — `tk_rope.tasm` (the kernel)

**Files:** Create `src/kernels/tk_rope.tasm`. Modify `src/kernels/registry.cpp`.

Kernel signature:
```
tk_rope(addr_X, addr_COS, addr_SIN, addr_ROTX, addr_Y):
  - r1 = addr_X     (27 input Word27s)
  - r2 = addr_COS   (27 cos_vec Word27s)
  - r3 = addr_SIN   (27 sin_vec Word27s)
  - r4 = addr_ROTX  (27 rotated_x Word27s)
  - r5 = addr_Y     (27 output Word27s)
```

Algorithm:
```
V0 = tvload(addr_X)        ; x
V1 = tvload(addr_COS)      ; cos_vec
V2 = tvload(addr_SIN)      ; sin_vec
V3 = tvload(addr_ROTX)     ; rotated_x

V4 = tvmul(V0, V1)         ; x * cos
V5 = tvmul(V3, V2)         ; rotated_x * sin
V6 = tvadd(V4, V5)         ; sum

tvstore V6 to addr_Y
halt
```

Saturation analysis: `x_int` bounded `[-9841, 9841]`, `cos_int`/`sin_int` bounded `[-OUT_SCALE, +OUT_SCALE]` where `OUT_SCALE = 9841`. Each `tvmul` peaks at `9841² ≈ 9.7e7` (fits int32). Sum of two such values fits int32 (peaks at `1.94e8`). No saturation. ✓

- [ ] **Step 1: `tests/test_kernel_rope.cpp`**

```cpp
#define DOCTEST_CONFIG_IMPLEMENT_WITH_MAIN
#include <doctest/doctest.h>
#include <ter/sim.hpp>
#include <ter/kernels.hpp>
#include <ter/numfmt.hpp>
#include <vector>
#include <cmath>
#include <random>
#include <algorithm>

#ifndef TER_KERNELS_DIR
#error "TER_KERNELS_DIR must be defined"
#endif

using namespace ter;

TEST_CASE("tk_rope rotates a 26-element pair-vector matching numpy reference") {
    constexpr int HEAD_DIM = 26;
    constexpr int N_PAIRS = HEAD_DIM / 2;
    constexpr int VEC_LANES = 27;
    constexpr int OUT_SCALE = 9841;
    constexpr int POS = 5;

    // Random input.
    std::vector<float> x(VEC_LANES, 0.0f);
    std::mt19937 rng(0x1337);
    std::uniform_real_distribution<float> dist(-2.0f, 2.0f);
    for (int i = 0; i < HEAD_DIM; ++i) x[i] = dist(rng);

    // Numpy reference: standard RoPE.
    std::vector<float> y_ref(VEC_LANES, 0.0f);
    for (int k = 0; k < N_PAIRS; ++k) {
        double freq = 1.0 / std::pow(10000.0, (2.0 * k) / double(HEAD_DIM));
        double angle = double(POS) * freq;
        double c = std::cos(angle);
        double s = std::sin(angle);
        double x0 = x[2 * k], x1 = x[2 * k + 1];
        y_ref[2 * k]     = static_cast<float>(x0 * c - x1 * s);
        y_ref[2 * k + 1] = static_cast<float>(x0 * s + x1 * c);
    }

    // Quantize x to format B.
    TritTensor xt = quantize(x.data(), {VEC_LANES}, 9);

    // Build cos_vec, sin_vec, rotated_x as int arrays (no per-tensor scale, OUT_SCALE-fixed).
    std::vector<int> cos_vec(VEC_LANES, 0);
    std::vector<int> sin_vec(VEC_LANES, 0);
    std::vector<int> rotated_x(VEC_LANES, 0);
    for (int k = 0; k < N_PAIRS; ++k) {
        double freq = 1.0 / std::pow(10000.0, (2.0 * k) / double(HEAD_DIM));
        double angle = double(POS) * freq;
        int c_int = static_cast<int>(std::round(std::cos(angle) * OUT_SCALE));
        int s_int = static_cast<int>(std::round(std::sin(angle) * OUT_SCALE));
        cos_vec[2 * k]     = c_int;
        cos_vec[2 * k + 1] = c_int;
        sin_vec[2 * k]     = s_int;
        sin_vec[2 * k + 1] = s_int;

        // rotated_x: lane (2k, 2k+1) = (-x[2k+1], x[2k]) using QUANTIZED x_int values
        int x0_int = static_cast<int>(xt.payload[2 * k].to_int());
        int x1_int = static_cast<int>(xt.payload[2 * k + 1].to_int());
        rotated_x[2 * k]     = -x1_int;
        rotated_x[2 * k + 1] = x0_int;
    }

    // Sim setup. Use addresses well above kernel-code region.
    Sim s(8192);
    KernelTable kt;
    install_default_kernels(s, kt, TER_KERNELS_DIR);
    KernelId id = kt.find("tk_rope");
    REQUIRE(id.valid);

    int x_addr = 1000, cos_addr = 1100, sin_addr = 1200, rotx_addr = 1300, y_addr = 1400;
    for (int i = 0; i < VEC_LANES; ++i) {
        s.mem().store_word(static_cast<size_t>(x_addr + i), xt.payload[i]);
        s.mem().store_word(static_cast<size_t>(cos_addr + i), Word27::from_int(cos_vec[i]));
        s.mem().store_word(static_cast<size_t>(sin_addr + i), Word27::from_int(sin_vec[i]));
        s.mem().store_word(static_cast<size_t>(rotx_addr + i), Word27::from_int(rotated_x[i]));
    }

    std::vector<int64_t> args = {x_addr, cos_addr, sin_addr, rotx_addr, y_addr, 0, 0};
    s.call_kernel(kt, id, args);

    // Read y_int back, recover floats.
    // y_int = x_int * cos_int + rotated_x_int * sin_int
    //       = (x / xt.scale) * (cos * OUT_SCALE) + (rotated_x_quant) * (sin * OUT_SCALE)
    //       ≈ ((x * cos - x_swapped * sin)) * OUT_SCALE / xt.scale
    // recovery = xt.scale / OUT_SCALE
    float recovery = xt.scale / static_cast<float>(OUT_SCALE);
    std::vector<float> y(VEC_LANES);
    for (int i = 0; i < VEC_LANES; ++i) {
        y[i] = static_cast<float>(s.mem().load_word(static_cast<size_t>(y_addr + i)).to_int()) * recovery;
    }

    // Compare on the first HEAD_DIM lanes only (lane 26 is unused / undefined).
    double max_rel = 0.0;
    for (int i = 0; i < HEAD_DIM; ++i) {
        double ref = y_ref[i];
        double got = y[i];
        double denom = std::max(1e-3, std::fabs(ref));
        double rel = std::fabs(got - ref) / denom;
        if (rel > max_rel) max_rel = rel;
    }
    CHECK(max_rel < 5e-2);

    CHECK(s.counters().get(Opcode::TVLOAD)  == 4);
    CHECK(s.counters().get(Opcode::TVMUL)   == 2);
    CHECK(s.counters().get(Opcode::TVADD)   == 1);
    CHECK(s.counters().get(Opcode::TVSTORE) == 1);
}
```

```markdown
## Lessons from `tk_rope` (host-prepared inputs unlock pure SIMD)

### When pair-wise / shuffle ops would need new opcodes, push to the host
RoPE's `(x[2k], x[2k+1]) → (x[2k]*c - x[2k+1]*s, x[2k]*s + x[2k+1]*c)` looks like it needs lane shuffle and pair-negate. Adding a `TVSWAP_PAIRS` opcode is one option; **building the rotated/duplicated vectors on the host before invoking the kernel is the cleaner one** for K3.

The host pays a small cost (a few loops over 27 elements) and the kernel becomes a tiny SIMD pipeline: `tvload x4 + tvmul x2 + tvadd + tvstore = 8 instructions`. Counter signature is exact and easy to verify.

This is the same architectural pattern as format-B matmul (host orchestrates K-tiling) and SwiGLU (host applies up multiplication). The principle: **complex setup belongs on the host; uniform SIMD math belongs in the kernel.**

### Recovery formula for pure-SIMD kernels
When the kernel does only linear ops (multiply + add) on quantized inputs and pre-scaled coefficients, the recovery is straightforward: divide by the input's scale and by the coefficient's scale once at the end. For RoPE: `recovery = xt.scale / OUT_SCALE`.

### Counter signature locks down kernel behaviour
RoPE's signature: `4 TVLOAD + 2 TVMUL + 1 TVADD + 1 TVSTORE`. Any deviation = bug. Pin all four exactly in the test.
```
