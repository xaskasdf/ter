# `ter` SiLU/SwiGLU Kernel Plan (F4-cont)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development.

**Goal:** Implement `tk_silu` — element-wise `silu(x) = x * sigmoid(x)` — as a ternary bytecode kernel using a host-generated sigmoid LUT. The full SwiGLU `y = silu(gate) * up` is the host-side composition of one `tk_silu` call followed by one `tvmul` (or another kernel call). End state: random gate + up vectors → SwiGLU via kernel + host → matches numpy reference within bounded relative error.

**Architecture:** Same per-lane LUT lookup pattern as `tk_softmax` (the kernel walks 27 iterations of a software loop because we lack lane-extract opcodes). The simplification vs softmax: only one LUT (sigmoid), no normalisation. Avoids the saturation pitfall of chained `tvmul`s by keeping multiplications to two-input only and recovering scale on the host.

**Tech Stack:** Same as plans 3 + 4.

**Spec:** `docs/superpowers/specs/2026-05-08-ter-design.md` §8 (kernel catalogue). Patterns: `docs/kernel-patterns.md`.

**Out of scope:** RoPE, attention. F5 bridge.

## Why SiLU instead of full SwiGLU

Full SwiGLU has three multiplications per element: `gate * sigmoid(gate) * up`. Chaining `tvmul gate*sigmoid` then `* up` saturates int32 in the second multiply (9841³ ≈ 9.5e11, way over int32_max ~2.1e9).

Cleanest solutions:
1. **Split into `tk_silu` + host-side tvmul** — what we do here. The host applies the per-tensor scale recovery, then a final `tvmul` against `up` (or invokes another kernel). No saturation in the kernel.
2. Add a `TVMUL_SHIFT` opcode that right-shifts after multiply. Future improvement, not this plan.
3. Extend `Vec` lanes to int64. Larger ISA churn, not justified yet.

Choosing (1) keeps the kernel simple and matches the established host-orchestration pattern.

## Task F4.8.1 — `gen_sigmoid_lut.py`

**Files:** Create `tools/gen_sigmoid_lut.py`.

The sigmoid LUT maps `idx ∈ [0, 255]` to `round(sigmoid(x_at_idx) * out_scale)` where `x_at_idx = (idx - 127.5) / 32` (input range `~ ±4` continuous units).

- [ ] **Step 1: Write `tools/gen_sigmoid_lut.py`**

```python
#!/usr/bin/env python3
"""Generates a 256-entry int32 sigmoid LUT for tk_silu."""
import argparse
import os
import numpy as np

def main():
    p = argparse.ArgumentParser()
    p.add_argument("--out-dir", default="build/lut_data")
    p.add_argument("--n-entries", type=int, default=256)
    p.add_argument("--out-scale", type=int, default=9841)
    a = p.parse_args()
    os.makedirs(a.out_dir, exist_ok=True)

    idx = np.arange(a.n_entries, dtype=np.float64)
    x = (idx - (a.n_entries - 1) / 2.0) / 32.0
    s = 1.0 / (1.0 + np.exp(-x))   # sigmoid in [0, 1]
    lut = np.round(s * a.out_scale).astype(np.int32)
    out_path = os.path.join(a.out_dir, "sigmoid_lut.bin")
    lut.tofile(out_path)
    with open(os.path.join(a.out_dir, "sigmoid_lut.meta"), "w") as f:
        f.write(f"n_entries={a.n_entries}\n")
        f.write(f"out_scale={a.out_scale}\n")
        f.write(f"x_step=0.03125\n")
        f.write(f"x_offset_idx={(a.n_entries - 1) / 2.0}\n")
    print(f"wrote {a.n_entries}-entry sigmoid LUT to {out_path}")
    print(f"sigmoid range covered: x in [{x[0]:.2f}, {x[-1]:.2f}], y in [{s[0]:.4f}, {s[-1]:.4f}]")

if __name__ == "__main__":
    main()
```

## Task F4.8.2 — `tk_silu.tasm`

**Files:** Create `src/kernels/tk_silu.tasm`. Modify `src/kernels/registry.cpp`.

Kernel signature:
```
tk_silu(addr_X, addr_Y, addr_LUT, x_scale_div):
  - r1 = addr_X (27 input Word27s)
  - r2 = addr_Y (27 output Word27 slots)
  - r3 = addr_LUT (256 sigmoid entries)
  - r4 = x_scale_div (x_int / r4 + 128 = idx)
```

Algorithm:
```
for k in 0..27:
  x_int = mem[addr_X + k]
  idx = clamp(x_int / x_scale_div + 128, 0, 255)
  s_int = mem[addr_LUT + idx]
  mem[scratch_S + k] = s_int

V0 = tvload(addr_X)
V1 = tvload(scratch_S)
V2 = tvmul(V0, V1)
tvstore V2 to addr_Y
halt
```

- [ ] **Step 1: Write `src/kernels/tk_silu.tasm`**

```
        ; tk_silu — 27-element silu(x) = x * sigmoid(x).
        ; Args:
        ;   r1 = addr_X
        ;   r2 = addr_Y
        ;   r3 = addr_LUT (256 sigmoid entries)
        ;   r4 = x_scale_div (x_int / r4 + 128 = idx)

        tloadi      r10, 1
        tloadi      r12, 27
        tloadi      r13, 128
        tloadi      r14, 700      ; scratch_S base
        tloadi      r11, 0        ; loop counter

silu_loop:
        tadd        r17, r1, r11
        tload       r18, r17        ; r18 = x_int

        tloadi      r19, 0          ; idx_signed
        tblt        r18, r0, neg_div
pos_div:
        tblt        r18, r4, after_div
        tsub        r18, r18, r4
        tadd        r19, r19, r10
        tjump       pos_div
neg_div:
        tadd        r21, r18, r4
        tblt        r0, r21, after_div
        tadd        r18, r18, r4
        tsub        r19, r19, r10
        tjump       neg_div
after_div:
        tadd        r19, r19, r13
        tblt        r19, r0, idx_zero
        tloadi      r22, 256
        tblt        r19, r22, idx_ok
        tsub        r19, r22, r10
        tjump       idx_ok
idx_zero:
        tloadi      r19, 0
idx_ok:
        tadd        r17, r3, r19
        tload       r23, r17        ; r23 = s_int

        tadd        r17, r14, r11
        tstore      r23, r17

        tadd        r11, r11, r10
        tblt        r11, r12, silu_loop

        tvload      v0, r1
        tvload      v1, r14
        tvmul       v2, v0, v1
        tvstore     v2, r2

        thalt
```

## Task F4.8.3 — End-to-end SwiGLU test vs numpy

**Files:** Create `tests/test_kernel_silu.cpp`. Modify `tests/CMakeLists.txt`.

The test exercises both the silu kernel AND the host-side `up` multiplication (full SwiGLU).

- [ ] **Step 1: `tests/test_kernel_silu.cpp`**

```cpp
#define DOCTEST_CONFIG_IMPLEMENT_WITH_MAIN
#include <doctest/doctest.h>
#include <ter/sim.hpp>
#include <ter/kernels.hpp>
#include <ter/numfmt.hpp>
#include <fstream>
#include <vector>
#include <cmath>
#include <random>
#include <algorithm>
#include <string>

#ifndef TER_KERNELS_DIR
#error "TER_KERNELS_DIR must be defined"
#endif

using namespace ter;

static std::vector<int32_t> read_i32(const std::string& path) {
    std::ifstream f(path, std::ios::binary);
    REQUIRE(f.is_open());
    f.seekg(0, std::ios::end);
    size_t n = static_cast<size_t>(f.tellg()) / 4;
    f.seekg(0);
    std::vector<int32_t> v(n);
    f.read(reinterpret_cast<char*>(v.data()), static_cast<std::streamsize>(n * 4));
    return v;
}

TEST_CASE("tk_silu matches numpy silu within bounded rel_err") {
    constexpr int N = 27;
    constexpr int N_ENTRIES = 256;
    constexpr int OUT_SCALE = 9841;

    std::vector<float> gate(N);
    std::mt19937 rng(0xDADA);
    std::uniform_real_distribution<float> dist(-2.0f, 2.0f);
    for (auto& v : gate) v = dist(rng);

    // Numpy reference: silu(g) = g * sigmoid(g)
    std::vector<float> y_ref(N);
    for (int i = 0; i < N; ++i) {
        double sig = 1.0 / (1.0 + std::exp(-double(gate[i])));
        y_ref[i] = static_cast<float>(double(gate[i]) * sig);
    }

    TritTensor gt = quantize(gate.data(), {N}, 9);

    auto sig_i32 = read_i32("lut_data/sigmoid_lut.bin");
    REQUIRE(sig_i32.size() == N_ENTRIES);
    std::vector<int> sigmoid_lut(sig_i32.begin(), sig_i32.end());

    float x_step = 0.0f;
    {
        std::ifstream meta("lut_data/sigmoid_lut.meta");
        REQUIRE(meta.is_open());
        std::string line;
        while (std::getline(meta, line)) {
            if (line.rfind("x_step=", 0) == 0) x_step = std::stof(line.substr(7));
        }
    }
    REQUIRE(x_step > 0.0f);

    Sim s(8192);
    KernelTable kt;
    install_default_kernels(s, kt, TER_KERNELS_DIR);
    KernelId id = kt.find("tk_silu");
    REQUIRE(id.valid);

    int x_addr = 100, y_addr = 200, lut_addr = 1000;
    for (int i = 0; i < N; ++i) s.mem().store_word(static_cast<size_t>(x_addr + i), gt.payload[i]);
    s.load_lut(lut_addr, sigmoid_lut);

    int64_t x_scale_div = static_cast<int64_t>(std::round(x_step / gt.scale));
    if (x_scale_div < 1) x_scale_div = 1;

    std::vector<int64_t> args = {x_addr, y_addr, lut_addr, x_scale_div, 0, 0, 0};
    s.call_kernel(kt, id, args);

    std::vector<int> y_int(N);
    for (int i = 0; i < N; ++i) y_int[i] = static_cast<int>(s.mem().load_word(static_cast<size_t>(y_addr + i)).to_int());

    // Recovery: y_int = gate_int * sigmoid_int = (gate / gt.scale) * (sigmoid * OUT_SCALE)
    // So  silu(gate) = y_int * gt.scale / OUT_SCALE.
    float recovery = gt.scale / static_cast<float>(OUT_SCALE);
    std::vector<float> y(N);
    for (int i = 0; i < N; ++i) y[i] = static_cast<float>(y_int[i]) * recovery;

    double max_rel = 0.0;
    for (int i = 0; i < N; ++i) {
        double ref = y_ref[i];
        double got = y[i];
        double denom = std::max(1e-3, std::fabs(ref));
        double rel = std::fabs(got - ref) / denom;
        if (rel > max_rel) max_rel = rel;
    }
    CHECK(max_rel < 1e-1);

    CHECK(s.counters().get(Opcode::TVMUL) == 1);
    CHECK(s.counters().get(Opcode::TVLOAD) == 2);
    CHECK(s.counters().get(Opcode::TVSTORE) == 1);
    // Per-lane scalar work: 27 reads + 27 lookups = 54+ tloads, 27 scratch tstores.
    CHECK(s.counters().get(Opcode::TLOAD) >= 54);
}

TEST_CASE("Full SwiGLU = silu(gate) * up via host composition") {
    constexpr int N = 27;
    constexpr int OUT_SCALE = 9841;

    std::vector<float> gate(N), up(N);
    std::mt19937 rng(0xFADE);
    std::uniform_real_distribution<float> dist(-2.0f, 2.0f);
    for (int i = 0; i < N; ++i) { gate[i] = dist(rng); up[i] = dist(rng); }

    std::vector<float> y_ref(N);
    for (int i = 0; i < N; ++i) {
        double sig = 1.0 / (1.0 + std::exp(-double(gate[i])));
        y_ref[i] = static_cast<float>(double(gate[i]) * sig * double(up[i]));
    }

    TritTensor gt = quantize(gate.data(), {N}, 9);

    auto sig_i32 = read_i32("lut_data/sigmoid_lut.bin");
    std::vector<int> sigmoid_lut(sig_i32.begin(), sig_i32.end());

    float x_step = 0.0f;
    {
        std::ifstream meta("lut_data/sigmoid_lut.meta");
        std::string line;
        while (std::getline(meta, line)) {
            if (line.rfind("x_step=", 0) == 0) x_step = std::stof(line.substr(7));
        }
    }

    Sim s(8192);
    KernelTable kt;
    install_default_kernels(s, kt, TER_KERNELS_DIR);
    KernelId id = kt.find("tk_silu");
    int x_addr = 100, y_addr = 200, lut_addr = 1000;
    for (int i = 0; i < N; ++i) s.mem().store_word(static_cast<size_t>(x_addr + i), gt.payload[i]);
    s.load_lut(lut_addr, sigmoid_lut);

    int64_t x_scale_div = std::max<int64_t>(1, static_cast<int64_t>(std::round(x_step / gt.scale)));
    std::vector<int64_t> args = {x_addr, y_addr, lut_addr, x_scale_div, 0, 0, 0};
    s.call_kernel(kt, id, args);

    // Recover silu values, then host-multiply by up.
    float recovery = gt.scale / static_cast<float>(OUT_SCALE);
    std::vector<float> y(N);
    for (int i = 0; i < N; ++i) {
        float silu = static_cast<float>(s.mem().load_word(static_cast<size_t>(y_addr + i)).to_int()) * recovery;
        y[i] = silu * up[i];
    }

    double max_rel = 0.0;
    for (int i = 0; i < N; ++i) {
        double ref = y_ref[i];
        double got = y[i];
        double denom = std::max(1e-3, std::fabs(ref));
        double rel = std::fabs(got - ref) / denom;
        if (rel > max_rel) max_rel = rel;
    }
    CHECK(max_rel < 1e-1);
}
```

```markdown
## Lessons from `tk_silu` (saturation-aware kernel design)

### Avoid chained `tvmul` saturation by splitting at host boundary
Full SwiGLU is `gate * sigmoid(gate) * up`. Doing all three multiplies in the kernel chains two `tvmul`s, the second of which saturates int32 (9841³ ≈ 9.5e11 > int32_max). The fix: kernel emits silu(gate) only; the host applies the per-tensor scale recovery and then multiplies by `up`. This mirrors the format-B matmul pattern where host-side scale recovery is the natural seam.

When designing future kernels, count the worst-case `tvmul` chain: `9841² = 9.7e7` fits in int32 (one chained multiply OK); `9841³ = 9.5e11` does not (two chained multiplies need scale-down between).

### Two TEST_CASEs per kernel: kernel-only and composed
For composable kernels (`silu` in SwiGLU, future `attention` in transformers), write two test cases:
1. Kernel output vs the function's direct numpy reference (e.g., `silu(gate) vs gate * sigmoid(gate)`).
2. Host composition vs the higher-level function (e.g., `silu(gate) * up vs swiglu(gate, up)`).

Both pass = kernel is correct AND composes correctly. Either failing pinpoints where the bug is.
```
